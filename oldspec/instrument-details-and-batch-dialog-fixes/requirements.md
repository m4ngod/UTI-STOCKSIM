# Requirements: instrument-details-and-batch-dialog-fixes

## Summary
修复 AgentsPanel 批量创建弹窗缺失导致的崩溃；实现标的详情页（含 L2 五档、日K 可缩放/可拖动、全体多策略散户汇总持仓饼图）以及“创建标的”弹窗三元联动计算（价格/流通股/市值）的实时推导逻辑。

## Scope
- UI 层：
  - AgentsPanelAdapter 缺失 `_open_batch_dialog` 方法导致点击“Batch Create…”报错。
  - 标的列表支持双击进入标的详情页。
  - 标的详情页包含：
    - L2 五档买卖报价与数量
    - 日K线图（鼠标/手势缩放、拖动，支持滚动到发行首日和最新日）
    - 全部“多策略散户”持仓合并后的持仓占比饼图
  - “创建标的”弹窗内：价格(元)、流通股(亿)、市值(亿) 三元联动推导，实时更新，修改任一字段自动计算另外一项（优先改动价格，若改动字段为价格则改动流通股）。
- 逻辑层：
  - 提供拉取标的详情（行情+K线+持仓统计）的查询接口；若已有服务则复用并补齐字段；无则最小实现。
  - 批量创建标的的服务入口（若已有则复用）。

## Out of Scope
- 实时撮合、行情源实现；可使用模拟或现有缓存数据。
- 高性能图形渲染优化；采用现有图表库能力即可。

## User Stories
- 作为用户，我在标的列表双击某个标的时，能进入该标的详情，查看五档、日K并缩放拖动到任意区间，同时看到多策略散户总体持仓占比饼图。
- 作为用户，我在“创建标的”弹窗输入价格/流通股/市值任意一个后，另一个会自动计算，无需手算，提高效率；当我修改价格字段时，以流通股为基准推导市值；当我修改非价格字段时，以价格为基准推导另一字段。
- 作为用户，我点击“Batch Create…”不会再崩溃，并能启动批量创建流程，同时在主界面看到进度更新。

## Functional Requirements
1) AgentsPanel 修复
- R1: 为 AgentsPanelAdapter 增加 `_open_batch_dialog()`，支持最小弹窗输入批量数量/类型，调用逻辑层接口开始创建，并监听进度事件；消除 AttributeError。

2) 标的详情页
- R2: 在标的列表中双击某条目打开“标的详情窗体/面板”。
- R3: 详情页显示五档（买1-买5、卖1-卖5，含价格与数量）。
- R4: 详情页显示日K线图，支持：
  - 拖动平移；
  - 滚轮缩放；
  - 最左可达发行首日，最右可达最新交易日。
- R5: 详情页显示“多策略散户总体”持仓占比饼图。

3) 创建标的弹窗三元联动
- R6: 弹窗包含价格(元)、流通股(亿)、市值(亿)三个可编辑输入框；实时联动。
- R7: 修改价格字段时，保持流通股不变，自动推导市值=价格×流通股；
- R8: 修改非价格字段（流通股或市值）时，优先修正价格字段：
  - 若改动流通股，价格=市值/流通股；
  - 若改动市值，价格=市值/流通股（保持流通股当前值）。
- R9: 数值变化立即在 UI 中反映，避免循环触发；支持小数精度与非法输入保护。

## Non-Functional Requirements
- N1: UI 更新在 100ms 内完成（本地数据）。
- N2: 兼容 headless 测试（PySide6 不可用时使用降级桩）。
- N3: 保持现有架构风格（Adapter + Service）。

## Acceptance Criteria
- AC1: 点击 AgentsPanel 的“Batch Create…”不再报错，能成功弹出对话框并调用服务，主界面进度标签更新。
- AC2: 双击标的项后弹出详情页，显示五档、日K（可缩放/拖动，边界到发行首日与最新日）、饼图。
- AC3: 创建标的弹窗三元联动按照 R6-R9 规则准确运行；边界/非法输入不会崩溃。

## Risks & Mitigations
- 图表库缺失或不稳定：采用可用的 PySide6 + matplotlib/pyqtgraph；headless 用桩绕过。
- 数据源不足：提供模拟数据或从已有服务聚合。

## Traceability
- Maps: R1→AC1；R2-R5→AC2；R6-R9→AC3；N1-N3→整体质量目标。
