# Product Overview

## Product Purpose
StockSim 是一个面向量化研究、策略验证与强化学习实验的多标的股票撮合与账户仿真平台。它聚合: 订单撮合(多簿)、风险控制、资金/持仓/借券账户体系、IPO 自动开盘、事件驱动前端展示、以及基于 PPO 的组合型强化学习环境。目标是在本地单机即可获得接近真实交易规则约束与成交细节的高保真模拟，缩短策略从构想到验证与迭代的周期，并为后续接入真实行情与实盘网关奠定结构基础。

解决的问题:
- 传统回测忽略撮合细节与买卖盘微观结构 → 还原盘口/阶段/节流的成交真实感。
- 策略、撮合、账户、风险分散在不同脚本中 → 以服务+事件总线解耦统一编排。
- 强化学习需要统一的多标的权重/动作分配接口 → 提供标准化 Env 与 PPO/LSTM 策略执行器。
- 卖空/IPO 等特殊场景常被简化 → 内置借券余额、自动开盘逻辑与回补闭环。

## Target Users
1. 量化研究员（需要快速验证微观结构敏感的策略如盘口择时、套利、冲击成本评估）。
2. 策略工程师 / 自营或私募开发（需要可扩展、多标的统一撮合与风险校验沙箱）。
3. 强化学习研究者（需要连续动作/组合再平衡/多资产权重分配环境）。
4. 教学 / 培训场景（需要可视化、可解释的撮合与账户演进过程）。
5. 高级个人投资者/开源贡献者（希望尝试插件化策略 / 指标 / 可视化扩展）。

痛点与需求:
- 重现性 (Determinism)
- 与真实交易阶段和撮合逻辑的贴近度
- 快速加载/切换多标的环境
- 策略→执行→成交→账户→事件→前端 的透明链路
- 可插拔 / 可裁剪 / 易重构

## Key Features
1. 多标的撮合引擎 (MatchingEngine + BookState) 支持动态注册 / 隔离盘口。
2. 事件驱动架构 (EventBus) 低耦合传播 SNAPSHOT / TRADE / ACCOUNT / ORDER 等事件。
3. 账户与持仓/借券模型：资金冻结、负仓持有、borrowed_qty 回补闭环。
4. 卖空与借券池 (LendingPool + RiskEngine 集成) 支持自动借券与归还。
5. IPO 集合竞价仅买单自动开盘逻辑外部化 (ipo_service)。
6. 快照节流 (ops_since_snapshot + 阈值) 降低 UI 与事件风暴。
7. 风险控制：资金、仓位、可借券、频率、日内限制（可扩展策略）。
8. 前端 PySide6 桌面应用：订单、盘口、成交、账户、图表+指标 多面板。
9. 指标与图表注册体系 (IndicatorRegistry) 支持动态扩展。
10. 强化学习环境 (EventTradingEnv) + PPO/LSTM 组合策略执行器。
11. Unit of Work + Repository + SQLAlchemy 持久化抽象与死锁重试。
12. 可配置撮合阶段 (集合竞价 / 连续竞价) 与时间压缩 SimClock。
13. 结构化日志与指标埋点（observability 目录，可扩展 Prom/Grafana）。
14. 模块化 services：订单 / 标的 / 账户 / 风险 / 借券 / 聚合条线清晰。

## Business Objectives
- 缩短策略原型验证周期 ≥ 50%。
- 提升微结构模拟可信度，减少实盘滑点/风控意外差异。
- 统一 RL 与经典规则策略执行渠道，降低双栈维护成本。
- 建立可扩展内核，后续平滑引入实时行情或撮合回放。
- 打造教学/演示友好栈，支持新成员 < 1 天完成上手回测。
- 促进开源协作：清晰分层 + 插件点降低贡献门槛。

## Success Metrics
- 回测/仿真吞吐: ≥ 5e5 订单事件/小时（单机基准数据集）。
- 撮合延迟中位数: < 200 µs (Python 级别内部逻辑，不含 I/O)。
- 策略迭代周期: 从想法→首个实测 < 2 小时（含数据装载）。
- RL 训练加速: 单日仿真压缩比 ≥ 480:1 (4 小时 → 30 秒)。
- 新开发者上手时间: 读完 README + 跑通示例 ≤ 0.5 天。
- 单元/集成测试覆盖: 初始 40%→阶段目标 70%。
- 稳定性: 压测 8 小时持续撮合 0 崩溃 / 0 死锁（≤1 次自动重试恢复允许）。
- 关键数据库查询 P95 < 30 ms；死锁重试成功率 ≥ 99%。

## Product Principles
1. Deterministic & Traceable：所有撮合/账户动作可重放、可追踪、可审计。
2. 分层解耦 (Separation of Concerns)：核心撮合、服务编排、前端展示、智能体严格分离。
3. Fail Fast + 可恢复：风险校验前置，错误早暴露，事务/重试机制保障一致性。
4. Extensibility by Design：策略、指标、风险规则、引擎扩展点显式暴露。
5. 最小必要复杂度：优先保持同步内核清晰性，再引入异步/分布式。
6. 观测优先 (Observability First)：日志、指标、事件使用统一语义，便于分析。
7. 安全与合规前置：输入验证、权限/隔离、借券/风控逻辑明确。

## Monitoring & Visibility
- Dashboard 类型: 桌面 PySide6 UI（后续可补 Web 只读仪表板）。
- 实时更新机制: EventBus → EventBridge → Qt Signals (批量节流 + UI 刷新)。
- 关键展示:
  - 行情快照(盘口/成交价/成交量)
  - 订单生命周期与状态变更
  - 账户资金/持仓/借券指标
  - 策略决策输出与执行记录
  - RL 训练回合收益 / 动作分布（后续面板）
- 可导出/分享: 计划支持结构化 CSV / JSON 回放脚本生成。
- 运维级指标（规划中）: 撮合延迟直方图、事件队列深度、数据库重试计数、内存占用、策略决策耗时。

## Future Vision
平台将在以下方向演进:

### Potential Enhancements
- Remote Access / Web 化: 轻量只读行情与策略监控界面 + Token 鉴权。
- 全链路回放: 日志/事件时序重放与“时间旅行”调试。
- 多进程/多线程撮合分区: 按 symbol 分片 + 共享只读元数据。
- 更细粒度风险模块: 动态保证金、成交冲击成本建模、风控脚本 DSL。
- 策略沙箱隔离: 资源限额、异常熔断、权限白名单。
- 强化学习扩展: 多任务 Multi-Env curriculum、指标特征自动提取。
- 分布式训练: 与 Ray / Accelerate 集成提升 RL 训练吞吐。
- 数据通道统一: 实盘行情 / 历史回放 / 合成数据的抽象层。
- 分层缓存与持久化优化: 热数据内存化 + 冷数据延迟加载。
- 多用户与权限: 账户作用域隔离、审计轨迹持久化。
